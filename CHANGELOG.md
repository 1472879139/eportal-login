# 更新日志

## v1.0.6 (2026-06-18)

### 新增功能

- **keep-alive 可视化倒计时** — 状态栏刷新按钮左侧新增"下次检测: M:SS"倒计时，启动后持续递减，归零时触发检测并自动重置。用户可随时看到距离下次自动检测还有多久

### 修复

- **已登录时刷新状态栏卡死** — 修复手动刷新后在已认证状态下状态栏消息停留在"正在刷新状态..."不更新的问题

## v1.0.5 (2026-06-18)

### 修复

- **运营商选错错误提示为编码** — `_friendly_error` 新增 `operator`、`account`、`product` 关键词匹配，运营商/套餐选错时不再显示 Base64 原文，而是映射为中文提示
- **浏览器注销后刷新仍显示已认证** — 新增 `_actually_disconnected` 标志追踪 UI 与实际认证状态不一致的场景。浏览器注销或被其他设备踢掉后，程序保持登录态以支持断线重连，同时状态栏提示"认证已断开，正在等待断线重连中"。该状态下点击注销只重置 UI，不发起真正的注销请求（因为实际已是注销状态）

### 改进

- **状态不一致提示** — keep-alive 检测到 UI 与实际状态不一致时，状态栏持续显示"认证已断开，正在等待断线重连中，若想立即重连，请退出程序重启或点击注销"，刷新按钮也会触发相同提示
- **注销按钮智能处理** — 状态不一致时注销按钮变为"重置 UI"按钮，只恢复登录表单可编辑状态，避免对已注销的会话发起无效请求

## v1.0.4 (2026-06-18)

### 修复

- **重启后"缺少网络参数无法注销"** — 修复两个导致网络参数丢失的根因：(1) 点击"保存设置"时 `_collect_config()` 不含 `cached_network_params`，直接覆盖配置文件导致参数被清空；(2) 首次安装或从未通过本程序登录过时，配置文件没有缓存参数，已认证状态下无法通过强制门户重定向获取

### 新增功能

- **本机参数兜底** — 当配置文件没有缓存的网络参数时，自动通过本机 IP 反查对应网卡 MAC，构造注销所需的最小参数集。注销请求实际只需 `wlan_user_ip` + `wlan_user_mac`（`wlan_ac_ip` / `wlan_ac_name` 固定为空），无需强制门户重定向
- **PowerShell 精确查 MAC** — 在存在 Hyper-V / VPN 等虚拟网卡时，`uuid.getnode()` 可能返回错误的 MAC 地址，现优先通过 `Get-NetAdapter` 按 IP 精确反查真实网卡 MAC，失败时降级到 `uuid.getnode()`

### 改进

- **"保存设置"合并写入** — 点击"保存设置"现在会先加载已有配置再合并写入，保留 `cached_network_params` 等内部字段不被覆盖
- **`logout()` 三层 fallback** — 注销时获取网络参数现在依次尝试：内存缓存 → 强制门户重定向 → 本机 IP/MAC 构造，确保尽可能不阻止用户注销

## v1.0.3 (2026-06-18)

### 新增功能

- **手动刷新状态** — 状态栏右侧新增"刷新"按钮，点击后立即检测当前认证状态并更新界面，无需等待定时器轮询。刷新仅查看状态，不触发自动重连
- **启动时自动登录** — 新增"启动时自动登录"复选框，勾选后程序启动检测到未认证状态时自动填写凭据完成登录。配合开机自启使用可实现无人值守自动联网

### 修复

- **刷新按钮卡在"正在刷新"** — 修复 `_on_status_update` 在状态未变时提前 return 导致消息未更新的问题，现刷新完成后始终显示确认消息
- **刷新误触发自动重连** — `_on_status_update` 新增 `allow_reconnect` 参数，刷新按钮传入 `False` 仅显示状态，keep-alive 定时器保持 `True` 自动重连
- **刷新破坏断线重连** — 修复刷新检测到断线时调用 `_apply_logged_out_state` 将 `_is_logged_in` 改为 `False`，导致 keep-alive 误认为用户从未登录而不再尝试重连。现刷新仅更新消息，不改变内部状态

## v1.0.2 (2026-06-18)

### 新增功能

- **记住密码选项** — 新增"记住密码"复选框，勾选后密码加密保存、下次启动自动填充；取消勾选后登录不保存密码，下次启动仅自动填充账号（类似 QQ 登录体验）
- **已登录锁定账号字段** — 登录认证后账号、密码、设备、运营商等所有输入控件变为不可编辑状态，注销后自动恢复
- **登录错误友好提示** — 账密错误时自动解码服务器返回的 Base64 错误信息，映射为中文提示：账号不存在 / 密码错误

### 改进

- **无条件记忆账号** — 无论是否勾选"记住密码"，账号始终自动保存并填充
- **配置向后兼容** — 升级用户默认启用"记住密码"，现有行为不变
- **VPN/代理友好提示** — 探测到所有地址均未触发校园网重定向时（如 VPN 开启），给出明确提示建议关闭 VPN 而非显示 HTTP 错误码
- **实时状态监控** — keep-alive 定时器现在始终运行状态检测，不再仅在"已登录"时才检查。未登录时以 30 秒间隔快速轮询，已登录后降为可配间隔。解决了: 1) 开机自启时网络延迟导致永久显示"未连接" 2) 在浏览器注销或被其他设备挤掉后状态不更新的问题
- **状态切换逻辑重构** — 登录/注销 UI 状态切换抽取为 `_apply_logged_in_state` / `_apply_logged_out_state` 复用，减少重复代码

### 修复

- **注销超时误报** — 修复注销请求因网络延迟超时时，提示"注销失败：timed out"但实际注销已成功的问题。现在注销超时后会自动检测认证状态来确认实际结果，避免误报
- **开机自启 VBS 脚本错误** — 修复勾选开机自启后重启弹出 VBScript 错误（800A0401 / 80070002）。根因为: 1) VBS 内嵌引号未按 `""` 转义 2) VBS 文件用 `encoding="utf-8"` 写入，含中文的路径被 VBScript 按系统编码（GBK）误读。修复: 引号正确转义 + 改用系统默认编码

### 移除

- **密码可见性切换** — 移除小眼睛（密码显示/隐藏）按钮，密码始终以遮罩显示，消除已保存密码被他人窥视的安全隐患

## v1.0.1 (2026-06-16)

### 新增功能

- **启动认证状态检测** — 程序启动时自动检测校园网是否已认证，已认证时直接显示"已连接"状态并可注销
- **弹窗提示** — 登录/注销成功或失败时弹出居中提示窗口，需点击"确定"关闭，重要信息不再错过
- **重复登录拦截** — 已处于认证状态时点击登录会弹出提示阻止提交，避免错误操作

### 改进

- **注销按钮智能可用** — 检测到已认证状态时注销按钮自动变为可点击，无需手动打开浏览器注销
- **断线重连优化** — keep-alive 定时器先探测认证状态，已在线则跳过，减少不必要的登录请求
- **网络参数持久化** — 登录成功后将网络参数保存到配置文件，支持跨会话注销
- **密码加密存储** — 基于本机 MAC 地址派生密钥加密密码，配置文件中不再明文显示

## v1.0.0 (2026-06-15)

CQUPT 校园网登录工具首次发布。

### 新增功能

- **可视化登录界面** — 基于 tkinter 的桌面窗口，无需接触命令行即可使用
- **一键登录/注销** — 连接校园网后点击登录完成认证，点击注销断开认证
- **设备伪装** — 支持 PC / Mobile 两种设备模式，用 Mobile 模式可实现双电脑共享账号上网
- **运营商选择** — 支持中国电信、中国移动、中国联通三种运营商
- **凭据记忆** — 登录时自动保存账号密码，下次打开自动填充
- **密码显示/隐藏** — 点击眼睛图标切换密码可见性
- **开机自启** — 勾选后在 Windows 启动文件夹创建静默启动脚本，开机自动运行
- **断线自动重连** — 默认每 5 分钟自动重连，间隔可自定义，保持持续在线
- **退出选项** — 关闭窗口时可选择先注销再退出，或直接退出保持在线
- **独立可执行文件** — PyInstaller 打包为单个 exe，无需安装 Python 环境
